## 2. 服务器规划

### 部署拓扑

```
                          ┌─────────────────────────────────────┐
                          │          Server 服务器               │
                          │                                     │
  用户浏览器 ──→ :8080 ──→│  Vite dev server (前端)             │
                          │    ↓ proxy /api → :8081              │
                          │    ↓ proxy /ws  → ws://:8081         │
                          │                                     │
                          │  Server (Spring Boot, 端口 8081)    │
                          │    └── H2 嵌入式数据库 (./data/ops)  │
                          │                                     │
                          └────────────┬────────────────────────┘
                                       │
                     HTTP 心跳上报       │  HTTP 部署指令/文件传输
                                       │
                          ┌────────────▼────────────────────────┐
                          │     Agent 服务器 (各业务机)          │
                          │     Agent (Spring Boot, 端口 2123)  │
                          │     心跳 → Server:8081               │
                          └─────────────────────────────────────┘
```

### 端口规划

| 服务 | 端口 | 说明 | 是否需要外部访问 |
|------|------|------|----------------|
| Vite dev server（前端） | **8080** | 用户浏览器访问入口 | 是（用户） |
| Server | 8081 | 后端 API 服务 | 是（Agent + 前端 proxy） |
| Agent | 2123 | 节点代理服务 | 否（仅连 Server） |

```bash
# 在本地打包（排除 .git 和 node_modules）
cd /path/to/easy-ops
tar czf easy-ops.tar.gz --exclude='.git' --exclude='node_modules' --exclude='target' --exclude='dist' .

scp easy-ops.tar.gz root@YOUR_SERVER:/opt/
```

## 5. Server 配置修改（重点）

### 5.1 修改 application.yml

**文件路径**：`backend/server/src/main/resources/application.yml`

需要修改以下配置项：

#### 5.1.1 端口：保持 8081

```yaml
server:
  port: 8081    # 保持 8081，前端 Vite proxy 转发到此端口
  servlet:
    context-path: /api
```

#### 5.1.2 数据库路径：改为绝对路径

```yaml
spring:
  datasource:
    url: jdbc:h2:/opt/easy-ops/backend/server/data/ops;MODE=MySQL
```

#### 5.1.3 数据目录：改为绝对路径

```yaml
server:
  path: /opt/easy-ops/backend/server/data
```

#### 5.1.4 CORS 配置：添加前端访问地址

**当前值**：
```yaml
cors:
  allowed-origins: "http://localhost:3000,http://localhost:5173"
```

**修改为**（前端通过 `http://内网IP:8080` 访问）：
```yaml
cors:
  allowed-origins: "http://localhost:3000,http://localhost:5173,http://YOUR_SERVER_IP:8080,http://127.0.0.1:8080"
```

> **重要**：`YOUR_SERVER_IP` 替换为服务器实际内网 IP。可用 `hostname -I` 查看。
> 如果前端和 Server 部署在同一台机器，至少加上 `http://127.0.0.1:8080` 和 `http://YOUR_IP:8080`。

#### 5.1.5 完整修改后的 application.yml

```yaml
server:
  port: 8081
  servlet:
    context-path: /api
    encoding:
      enabled: true
      charset: UTF-8
      force: true
  max-http-request-size: 10MB
  max-http-response-size: 10MB
  tomcat:
    max-swallow-size: 10MB
    accesslog:
      enabled: true
      pattern: '%h %l %u %t "%r" %s %b'
  # ===== 部署修改：改为绝对路径 =====
  path: /opt/easy-ops/backend/server/data
  heart-second: 5
  offline-second: 90

spring:
  application:
    name: ops-server
  profiles:
    active: dev
  datasource:
    driver-class-name: org.h2.Driver
    # ===== 部署修改：改为绝对路径 =====
    url: jdbc:h2:/opt/easy-ops/backend/server/data/ops;MODE=MySQL
    username: sa
    password:
  h2:
    console:
      enabled: ${H2_CONSOLE_ENABLED:false}
      path: /h2-console
  sql:
    init:
      mode: always
      schema-locations: classpath:db/schema.sql
  servlet:
    multipart:
      max-file-size: 500MB
      max-request-size: 500MB
  jackson:
    date-format: yyyy-MM-dd HH:mm:ss
    time-zone: Asia/Shanghai
    default-property-inclusion: non_null

jwt:
  secret: ${JWT_SECRET:}
  expire-ms: 86400000

mybatis:
  mapper-locations: classpath:mapper/*.xml
  configuration:
    map-underscore-to-camel-case: true
    log-impl: org.apache.ibatis.logging.slf4j.Slf4jImpl

logging:
  level:
    root: INFO
    com.ops.server: INFO
    com.ops.server.mapper: INFO
  file:
    name: ${server.path:./data}/logs/server.log
  logback:
    rollingpolicy:
      max-file-size: 100MB
      max-history: 30

# ===== 部署修改：添加生产环境域名 =====
cors:
  allowed-origins: "http://YOUR_SERVER_IP:8080,http://127.0.0.1:8080,http://localhost:3000,http://localhost:5173"

ai:
  apiKey: ${AI_API_KEY:}

websocket:
  max-connections: 1000
```

### 5.2 必须设置的环境变量

| 环境变量 | 必填 | 说明 | 示例 |
|---------|------|------|------|
| `JWT_SECRET` | **是** | JWT 密钥，至少 32 个字符 | `export JWT_SECRET='easy-ops-prod-secret-key-2026-xxxxx'` |
| `AI_API_KEY` | 否 | AI 分析 API 密钥（不使用 AI 功能可不设） | `export AI_API_KEY='sk-xxxxx'` |

> **SecurityConfig.java 启动校验**：如果 `JWT_SECRET` 未设置或长度 < 32 字符，Server 将拒绝启动！

### 5.3 启动命令（带环境变量）

```bash
cd /opt/easy-ops/backend/server

# 设置环境变量
export JWT_SECRET='easy-ops-prod-jwt-secret-2026-random-string-here'

# 启动 Server
nohup java -jar target/ops-platform-server-1.0.0-SNAPSHOT.jar \
  > /opt/easy-ops/backend/server/data/logs/server.log 2>&1 &

echo "Server PID: $!"
```

## 6. Agent 配置修改

### 6.1 修改 application.yml

**文件路径**：`backend/agent/src/main/resources/application.yml`

需要修改的配置项：

#### 6.1.1 Server 地址：指向实际 Server 内网 IP

```yaml
agent:
  server-url: http://YOUR_SERVER_IP:8081/api
```

#### 6.1.2 完整修改后的 application.yml

```yaml
server:
  port: 2123
  servlet:
    context-path: /api

spring:
  servlet:
    multipart:
      max-file-size: 500MB
      max-request-size: 500MB

agent:
  # ===== 部署修改：指向实际 Server 地址 =====
  server-url: http://YOUR_SERVER_IP:8081/api
  # ===== 部署修改：设置 Agent 认证 Token =====
  token: ${AGENT_TOKEN:}
  node-name: ${AGENT_NODE_NAME:default-node}
  data-path: /app/data
  check-interval: 30
  host-ip: ${AGENT_HOST_IP:}
  host-port: ${AGENT_HOST_PORT:0}

logging:
  level:
    com.ops.agent: INFO
  file:
    name: ./data/logs/agent.log
```

### 6.2 必须设置的环境变量

| 环境变量 | 必填 | 说明 | 示例 |
|---------|------|------|------|
| `AGENT_TOKEN` | **是** | Agent 认证 Token（需在 Server 端注册节点时一致） | `export AGENT_TOKEN='agent-token-1'` |
| `AGENT_NODE_NAME` | **是** | 节点名称（在 Server 端注册时使用） | `export AGENT_NODE_NAME='node-prod-1'` |
| `AGENT_SERVER_URL` | 否 | 可选覆盖 Server 地址 | `export AGENT_SERVER_URL='http://YOUR_SERVER_IP:8081/api'` |
| `AGENT_HOST_IP` | 否 | Agent 机器实际 IP（可选，心跳自动检测） | `export AGENT_HOST_IP='192.168.1.100'` |
| `AGENT_HOST_PORT` | 否 | Agent 机器实际端口 | `export AGENT_HOST_PORT='2123'` |

### 6.3 启动命令

```bash
cd /opt/easy-ops/backend/agent

export AGENT_TOKEN='agent-token-1'
export AGENT_NODE_NAME='node-prod-1'
export AGENT_SERVER_URL='http://YOUR_SERVER_IP:8081/api'

nohup java -jar target/ops-platform-agent-1.0.0-SNAPSHOT.jar \
  > ./data/logs/agent.log 2>&1 &

echo "Agent PID: $!"
```

## 7. H2 数据库说明（重点）

### 7.1 你不需要手动创建 H2 数据库

**H2 是嵌入式数据库**，不需要安装任何数据库服务。Spring Boot 启动时会自动：

1. 创建 `data/` 目录（如果不存在）
2. 创建 `data/ops.mv.db` 数据库文件（如果不存在）
3. 执行 `db/schema.sql` 初始化脚本（因为配置了 `spring.sql.init.mode: always`）
4. 创建所有表、索引、默认数据（管理员账号等）

**你只需要确保**：
- `data/` 目录有写权限
- `JWT_SECRET` 环境变量已设置（否则 Server 不会启动，数据库也不会被创建）

### 7.2 schema.sql 脚本内容（与代码完全一致）

## 8. 前端部署（Vite dev server）

前端使用 Vite dev server 运行，通过 `vite.config.ts` 中的 proxy 配置自动将 `/api` 和 `/ws` 请求转发到 Server:8081，**无需 Nginx**。

### 8.1 修改 vite.config.ts

**文件路径**：`frontend/vite.config.ts`

修改端口为 8080，并确认 proxy 配置：

```typescript
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import { resolve } from 'path'

export default defineConfig({
  plugins: [vue()],
  resolve: {
    alias: {
      '@': resolve(__dirname, 'src')
    }
  },
  server: {
    // ===== 部署修改：端口改为 8080，监听所有地址 =====
    port: 8080,
    host: '0.0.0.0',    // 允许外部访问（默认仅 localhost）
    proxy: {
      // API 请求代理 → Server:8081
      '/api': {
        target: 'http://localhost:8081',
        changeOrigin: true,
        ws: true
      },
      // WebSocket 代理 → Server:8081
      '/ws': {
        target: 'ws://localhost:8081',
        changeOrigin: true,
        ws: true,
        rewrite: (path) => path
      },
    }
  },
  build: {
    outDir: 'dist',
    sourcemap: false
  }
})
```

**关键配置说明**：
- `host: '0.0.0.0'`：允许外部 IP 访问（默认 Vite 只监听 127.0.0.1）
- `proxy`：自动将 `/api` 和 `/ws` 请求转发到 Server:8081，前端代码无需任何修改

### 8.2 前端 WebSocket 连接方式（无需修改）

前端代码中 WebSocket 连接使用相对路径：

```typescript
// ConsoleView.vue
const wsProtocol = location.protocol === 'https:' ? 'wss:' : 'ws:'
ws = new WebSocket(`${wsProtocol}//${location.host}/api/ws/console?${params}`)
```

这意味着 WebSocket 会自动连接到当前浏览器访问的域名和端口（即 `http://YOUR_IP:8080`），由 Vite proxy 转发到 Server:8081。**无需修改前端代码**。

## 9. 启动脚本

### 9.1 Server 启动脚本

```bash
cat > /opt/easy-ops/backend/start-server.sh << 'SCRIPT'
#!/bin/bash
# EasyOps Server 启动脚本
set -e

BASE_DIR="/opt/easy-ops/backend"
JAR="$BASE_DIR/server/target/ops-platform-server-1.0.0-SNAPSHOT.jar"
DATA_DIR="$BASE_DIR/server/data"
LOG_FILE="$DATA_DIR/logs/server.log"

# 检查 JAR 是否存在
if [ ! -f "$JAR" ]; then
    echo "[ERROR] JAR 文件不存在，请先编译：cd $BASE_DIR && mvn clean package -DskipTests"
    exit 1
fi

# 创建数据目录
mkdir -p "$DATA_DIR/logs"

# ===== 必须设置的环境变量 =====
export JWT_SECRET="${JWT_SECRET:-easy-ops-prod-jwt-secret-2026-secure-random-string}"

# 可选环境变量
# export AI_API_KEY="your-ai-api-key"

echo "=========================================="
echo "  EasyOps Server"
echo "  端口: 8081"
echo "  数据目录: $DATA_DIR"
echo "  日志: $LOG_FILE"
echo "=========================================="

# 启动 Server（后台运行）
cd "$BASE_DIR/server"
nohup java -jar "$JAR" > "$LOG_FILE" 2>&1 &
PID=$!

echo "Server PID: $PID"
echo "等待启动..."
sleep 5

# 检查是否启动成功
if kill -0 $PID 2>/dev/null; then
    echo "[OK] Server 启动成功"
    echo "API 地址: http://localhost:8081/api/"
    echo "停止命令: kill $PID"
else
    echo "[ERROR] Server 启动失败，查看日志："
    tail -30 "$LOG_FILE"
    exit 1
fi
SCRIPT

chmod +x /opt/easy-ops/backend/start-server.sh
```

## 12. 常见问题排查

### Q1: Server 启动失败 — "jwt.secret is not configured"

**原因**：未设置 `JWT_SECRET` 环境变量

**解决**：
```bash
export JWT_SECRET='your-random-secret-at-least-32-characters'
# 然后重新启动
```

### Q2: Server 启动失败 — "JWT_SECRET must be at least 32 characters"

**原因**：`JWT_SECRET` 长度不足 32 字符

**解决**：使用更长的密钥
```bash
export JWT_SECRET='easy-ops-prod-jwt-2026-$(openssl rand -hex 16)'
```

### Q3: 前端页面空白 / API 请求 404

**原因**：Vite proxy 未正确配置，或前端未启动

**解决**：
```bash
# 检查前端是否运行
pgrep -f "vite"

# 检查 vite.config.ts 中 proxy 配置是否正确
cat /opt/easy-ops/frontend/vite.config.ts | grep -A 10 proxy
```

### Q4: WebSocket 连接失败（控制台无法连接）

**原因**：Vite proxy 未配置 WebSocket，或前端 host 未设置为 0.0.0.0

**解决**：
- 确认 `vite.config.ts` 中 proxy 的 `ws: true`
- 确认 `server.host: '0.0.0.0'`

### Q5: Agent 无法连接 Server

**原因**：`AGENT_SERVER_URL` 配置错误，或防火墙未开放 8081 端口

**解决**：
```bash
# 检查 Agent 日志
tail -50 /opt/easy-ops/backend/agent/data/logs/agent.log

# 测试连通性
curl http://YOUR_SERVER_IP:8081/api/nodes/heartbeat
```

### Q6: 文件上传失败 — "Maximum upload size exceeded"

**原因**：Spring Boot 文件大小限制

**解决**：确认 `application.yml` 中：
```yaml
spring:
  servlet:
    multipart:
      max-file-size: 500MB
      max-request-size: 500MB
```

### Q7: H2 数据库文件损坏

**解决**：
```bash
# 停止 Server
kill $(pgrep -f "ops-platform-server")

# 删除损坏的数据库文件（会丢失所有数据）
rm /opt/easy-ops/backend/server/data/ops.mv.db
rm /opt/easy-ops/backend/server/data/ops.trace.db

# 重新启动 Server（会自动从 schema.sql 重建）
bash /opt/easy-ops/backend/start-server.sh
```

### Q8: 前端 CORS 跨域错误

**原因**：CORS 配置中的域名与实际访问地址不匹配

**解决**：检查 `application.yml` 中 `cors.allowed-origins` 是否包含实际访问地址，例如：
```yaml
cors:
  allowed-origins: "http://192.168.1.100:8080,http://127.0.0.1:8080"
```

### Q9: 外部无法访问前端（浏览器打不开页面）

**原因**：Vite 默认只监听 localhost

**解决**：确认 `vite.config.ts` 中设置了 `host: '0.0.0.0'`
