# 知识管理模块重新设计 PRD

> **版本**：v2.0（全面替换 v1.0）
> **作者**：许清楚（产品经理）
> **日期**：2026-06-27
> **对标产品**：有道云笔记
> **技术栈**：Vue 3 + Ant Design Vue + TypeScript（前端） | Spring Boot 2.7.18 + H2（后端）

---

## 1. 产品目标

**将 EasyOps 知识库从"只能存 Markdown 的简陋文本框"升级为"团队内云笔记系统"，让运维和开发人员像用有道云笔记一样高效地沉淀、协作和检索运维知识。**

三个核心目标：

| # | 目标 | 量化标准 |
|---|------|---------|
| G1 | **编辑体验与有道云笔记持平** | 富文本 + Markdown 双模式编辑，支持表格/代码块/图片/流程图，编辑器响应 < 200ms |
| G2 | **协作能力从"排队等待"升级为"实时共创"** | 多人可同时编辑同一文档，编辑冲突 < 5s 内自动合并，评论支持 @提及/回复/内联批注 |
| G3 | **知识检索从"翻文件夹"升级为"一搜即达"** | 全文搜索 + 标签系统 + 高级筛选，搜索结果 < 1s，命中标题/正文/标签 |

## 2. 用户故事

### US-1：运维人员撰写故障处理文档

> 作为运维人员张三，我想在富文本编辑器中撰写故障处理文档（含截图、命令输出、对比表格），以便团队其他成员能直观阅读而非面对一坨纯 Markdown 源码。

**验收标准**：编辑器支持富文本模式，可直接粘贴截图、插入表格、代码块语法高亮；一键切换 Markdown 模式，两种模式内容互通。

### US-2：多人协作编辑部署手册

> 作为运维组长李四，我想和组员王五同时编辑同一份"微服务部署手册"，我改第三章、他改附录，互不阻塞，实时看到彼此的修改位置，以便快速完成文档更新。

**验收标准**：打开文档时可见在线协作者光标位置；两人同时编辑不同段落时不产生冲突；5s 内远端修改可见；退出编辑后自动同步。

### US-3：开发人员查阅并评论运维知识

> 作为开发人员赵六，我想在阅读"OOM 故障诊断报告"时对某段内容直接批注提问（"这个 JVM 参数改过吗？"），并 @运维人员张三，以便精准沟通而非在 IM 里翻找上下文。

**验收标准**：支持选中文字后添加内联批注；批注显示在原文右侧浮动；@提及后被提及者收到通知；批注可回复形成讨论线程。

### US-4：新人快速搜索运维知识

> 作为刚入职的运维新人，我想搜索"tm-server 启动失败"关键词，立刻看到所有相关故障案例、FAQ、配置说明文档，并按标签"故障案例"进一步筛选，以便快速定位解决方案。

**验收标准**：搜索框输入关键词后 < 1s 返回结果；结果高亮命中片段；支持按标签、分类、时间范围等高级筛选；搜索结果可点击直接跳转文档。

### US-5：管理员管理笔记本与文档权限

> 作为运维平台管理员，我想创建"生产环境运维"笔记本（含图标、多层级子分类），将一批文档拖拽归类，并设置该笔记本对开发团队只读、对运维团队可编辑，以便控制知识访问权限。

**验收标准**：笔记本支持 3 层以上嵌套；支持拖拽排序/移动文档；分类支持自定义图标；权限可按角色/人员设置查看/编辑/管理三个等级。

### US-6：查看版本历史并回滚

> 作为文档作者张三，我想查看"部署手册"的版本历史，对比当前版本与上一版本的差异（新增/删除内容高亮标注），一键回滚到上一版本，以便修复误删内容。

**验收标准**：版本列表展示每个版本的编辑人、时间、变更摘要；版本对比视图以红绿标记增删内容；一键回滚后生成新版本（而非覆盖历史）。

### US-7：使用运维模板快速创建文档

> 作为运维人员，我想从"故障报告"、"部署检查清单"、"交接文档"等预设模板快速创建文档，以便不用每次从头排版，直接填内容。

**验收标准**：新建文档时可选择模板；模板包含预置标题结构、表格骨架、提示占位符；支持自定义保存为个人模板。

## 4. 核心功能模块说明

### 4.1 文档编辑器（富文本 + Markdown 双模式）

**当前问题**：纯 textarea，无格式化能力，粘贴截图只显示 URL 文本。

**目标设计**：

| 特性 | 富文本模式 | Markdown 模式 |
|------|-----------|--------------|
| 输入方式 |所见即所得，类似 Word | 左侧源码 + 右侧实时预览 |
| 切换 | 工具栏按钮一键切换 | 工具栏按钮一键切换 |
| 内容互通 | 富文本内容自动转为 MD 存储 | MD 渲染后转为富文本 DOM |
| 工具栏 | 格式（标题/粗斜/删除线）、插入（表格/图片/链接/代码块/分割线）、协作（评论批注入口）、模板 | MD 语法快捷插入 + 预览同步滚动 |
| 快捷键 | Ctrl+B/I/K/S 等 | 同上 |
| 图片 | 粘贴截图 → 自动上传 → 内嵌预览 | `![](url)` 语法，预览区实时渲染 |
| 代码块 | 选择语言 → 高亮渲染 | `` ```lang `` 语法，预览区高亮 |
| 表格 | 可视化增删行列、拖拽调整列宽 | MD 表格语法，预览区渲染 |

**技术选型建议**：
- 富文本引擎：Tiptap（基于 ProseMirror，Vue 3 友好，支持协作扩展）
- Markdown 引擎：保留 v-md-editor 或切换到 Milkdown（基于 ProseMirror，可与 Tiptap 共享底层）
- 协作层：Tiptap Collaboration 扩展 + Yjs（CRDT 库）

**数据存储**：文档内容统一以 Markdown 格式存储在 `kb_document.content`，富文本模式编辑时前端内部做 DOM↔MD 转换。

### 4.2 文档树 / 笔记本管理

**当前问题**：只有新建分类，无编辑/删除/拖拽排序/图标，无层级概念。

**目标设计**：

```
笔记本树（左侧侧栏 260px）
├── 📁 生产环境运维          ← 自定义图标 + 颜色
│   ├── 📋 故障案例          ← 二级分类
│   │   ├── 🔥 严重故障      ← 三级分类
│   │   └── ⚡ 一般故障
│   ├── 📋 部署手册
│   └── 📋 配置说明
├── 📁 开发交接文档
│   ├── 📋 API 说明
│   └── 📋 架构设计
├── ⭐ 收藏                 ← 虚拟分类（自动聚合收藏文档）
└── 🕐 最近编辑             ← 虚拟分类（自动聚合最近编辑文档）
```

**交互细节**：
- 拖拽排序：分类和文档均可拖拽调整顺序和归属（树形拖拽，支持跨层级）
- 右键菜单：分类 → 新建子分类/编辑/删除/更换图标/排序；文档 → 移动/复制/删除/收藏
- 新建分类弹窗：名称 + 图标选择器（预设图标库 ≥ 20） + 父分类选择
- 空分类删除：分类下有文档时提示"先移动文档再删除"
- 展开/折叠：全部展开/全部折叠按钮；单个分类点击展开/折叠

---

### 4.3 实时协作编辑

**当前问题**：悲观锁（一个人编辑时其他人只能等待），协作体验极差。

**目标设计**：

| 机制 | 说明 |
|------|------|
| **算法** | Yjs CRDT（Conflict-free Replicated Data Type），无需中心化锁 |
| **传输** | WebSocket `/ws/kb-collab/{documentId}`，前端通过 Yjs WebSocket provider 连接 |
| **光标感知** | 每个编辑者分配随机颜色，光标位置和选区实时同步显示 |
| **冲突合并** | CRDT 算法自动合并，不同段落修改无冲突；同一段落并发修改按字符级合并 |
| **离线支持** | 断线后本地继续编辑（Yjs 本地状态），重连后自动同步 |
| **人数上限** | 同一文档同时编辑上限 5 人（可配置），超过时新加入者进入只读+评论模式 |
| **在线列表** | 文档顶部显示当前在线编辑者头像列表 + 编辑状态标识 |

**流程**：

```
用户打开文档 → WS 连接 collab 房间 → 接收当前文档 Yjs Doc 状态
→ 前端 Tiptap 加载 Yjs Doc → 用户编辑产生本地操作
→ Yjs 自动合并+发送到 WS → 其他协作者实时接收并渲染
→ 用户关闭文档 → WS 断连 → 服务端保存最终状态为新版本
```

**与现有锁机制的过渡**：
- P0 阶段：保留编辑锁作为 fallback（当 CRDT 连接失败时降级为锁模式）
- 后续：CRDT 稳定后移除锁机制，`kb_document_lock` 表废弃

### 4.4 评论与讨论

**当前问题**：最简陋的文本评论，无 @提及、无回复、无点赞。

**目标设计**：

**两种评论形式**：

| 形式 | 位置 | 用途 |
|------|------|------|
| **文档评论** | 文档底部评论区 | 对整篇文档的讨论、提问、反馈 |
| **内联批注** | 富文本编辑器右侧浮动 | 对选中文字的精准提问/标注 |

**文档评论功能**：
- 评论输入框支持 Markdown 子集（粗体/链接/代码/图片）
- @提及：输入 `@` 弹出团队成员下拉列表，选择后生成 `@张三` 链接
- 回复：每条评论下方可回复，形成嵌套线程（最多 3 层）
- 点赞：每条评论右侧 👍 按钮，显示点赞数
- 通知：被 @提及者收到站内通知 + WS 实时推送
- 排序：按时间/点赞数排序

**内联批注功能**：
- 富文本模式：选中文字 → 工具栏"批注"按钮 → 右侧浮动批注面板
- 批注面板显示：批注文字高亮、批注内容、回复线程
- 批注标记在文中以黄色高亮+编号显示
- 批注列表可折叠/展开

---

### 4.5 文档共享与权限

**当前问题**：无权限控制，所有人都能编辑所有文档。

**目标设计**：

| 权限等级 | 能力 |
|---------|------|
| **查看** | 查看文档内容、评论、版本历史，不可编辑 |
| **编辑** | 查看 + 编辑内容 + 添加评论/批注 |
| **管理** | 编辑 + 删除文档 + 管理权限 + 移动文档 |

**权限分配粒度**：
- 笔记本级别：设置整个笔记本的默认权限（继承到子分类和文档）
- 文档级别：可单独覆盖笔记本默认权限
- 按人员分配：指定具体用户的权限等级
- 按角色分配：运维组/开发组/管理员组的默认权限

**外链分享**：
- 生成分享链接：可选设置访问密码、有效期（1天/7天/永久）
- 外链访问者：只读模式，不可评论/编辑
- 分享管理：文档作者可查看/取消已生成的外链

**与项目关联**：
- 笔记本可关联项目（`project_id`），项目成员自动获得查看权限
- 非 项目成员访问项目关联笔记本时提示"无权限"

### 4.6 版本历史与回滚

**当前问题**：后端有版本数据但前端无对比界面。

**目标设计**：

**版本列表页**：

```
版本历史面板（右侧抽屉 400px）
┌──────────────────────────────────────┐
│  📄 tm-server 故障处理记录           │
│  ────────────────────────────────     │
│  v12  张三  2026-06-27 14:30  [当前] │
│       "新增 JVM 参数调优章节"         │
│  v11  李四  2026-06-26 09:15         │
│       "修正部署步骤顺序"               │
│  v10  张三  2026-06-25 16:00         │
│       "添加 OOM 诊断结论"             │
│  ...                                 │
│                                      │
│  [对比 v11 ↔ v12]  [回滚到 v11]     │
└──────────────────────────────────────┘
```

**版本对比 Diff 视图**：
- 左侧：旧版本内容（删除内容红色高亮 + 删除线）
- 右侧：新版本内容（新增内容绿色高亮）
- 支持选择任意两个版本对比
- 支持逐段/全文切换

**回滚机制**：
- 一键回滚到选定版本 → 生成新版本号（当前版本号+1），内容为回滚版本的内容
- 回滚操作记录在版本变更摘要中："回滚至 v11"
- 不覆盖任何历史版本

### 4.7 搜索与标签

**当前问题**：简单关键词 LIKE 搜索，无标签系统，无高级筛选。

**目标设计**：

**搜索入口**：
- 顶部全局搜索栏（快捷键 Ctrl+Shift+F）
- 分类树内部筛选框
- 文档列表页筛选栏

**搜索能力**：

| 能力 | 说明 |
|------|------|
| 全文搜索 | 搜索标题 + 正文内容，结果高亮命中片段 |
| 标签搜索 | `#标签名` 搜索特定标签文档 |
| 分类筛选 | 指定分类范围内搜索 |
| 时间范围 | 创建时间/更新时间范围筛选 |
| 作者筛选 | 按文档作者筛选 |
| 排序 | 按相关性/更新时间/浏览量排序 |
| 高级语法 | 双引号精确匹配、`-` 排除关键词、`*` 通配符 |

**标签系统**：
- 文档编辑时可添加 ≤ 10 个标签
- 标签选择器：下拉已有标签 + 输入创建新标签
- 标签颜色：每个标签可设置颜色（预设 8 种颜色）
- 标签侧栏：分类树下方显示标签列表，点击标签筛选文档
- 标签管理：管理员可合并/删除标签

**搜索索引**：
- P0：升级 H2 全文索引（H2 FT 索引）替代 LIKE 搜索
- P1：后续可升级为嵌入式 Lucene 索引提升性能

### 4.8 模板系统

**当前问题**：无模板，每次从空白文档开始。

**目标设计**：

**预置模板**（ ≥ 5 个运维常用模板）：

| 模板名 | 结构 | 适用场景 |
|--------|------|---------|
| 故障报告 | 故障概述 → 影响范围 → 根因分析 → 解决方案 → 验证结果 → 预防措施 | 记录故障处理全过程 |
| 部署检查清单 | 部署前检查（表格）→ 部署步骤（列表）→ 部署后验证（表格）→ 回滚方案 | 规范化部署流程 |
| 交接文档 | 系统概述 → 架构说明 → 关键配置 → 常见问题 → 联系方式 | 开发→运维交接 |
| 配置变更记录 | 变更原因 → 变更内容（对比表格）→ 影响评估 → 执行步骤 → 验证结果 | 配置变更审批记录 |
| FAQ | 问题 → 答案（可折叠）→ 相关文档链接 | 运维常见问题汇总 |

**交互**：
- 新建文档时弹窗选择"空白文档"或"从模板创建"
- 模板创建后可自由修改，与普通文档无异
- 管理员可将任何已发布文档"保存为模板"
- 模板管理页：查看/编辑/删除预置和个人模板

**目标设计**：
