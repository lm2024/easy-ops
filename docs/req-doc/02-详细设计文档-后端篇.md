## 二、数据库表详细设计

共 7 张核心表，全部使用 MyBatis XML 映射，不使用 JPA/ORM 注解。

node_info 表 (节点信息表)
  id BIGINT PRIMARY KEY AUTO_INCREMENT
  name VARCHAR(100) UNIQUE NOT NULL
  ip VARCHAR(50) NOT NULL
  port INT DEFAULT 2123
  token VARCHAR(64) NOT NULL
  status TINYINT DEFAULT 0 (0=离线, 1=在线)
  os_info VARCHAR(200)
  java_version VARCHAR(50)
  last_heartbeat BIGINT
  create_time BIGINT
  update_time BIGINT
  索引: UNIQUE KEY uk_name (name), INDEX idx_status (status)

project_info 表 (项目管理表)
  id BIGINT PRIMARY KEY AUTO_INCREMENT
  name VARCHAR(100) UNIQUE NOT NULL
  node_ids VARCHAR(1000) (逗号分隔的节点ID列表)
  start_script TEXT (启动脚本)
  stop_script TEXT (停止脚本)
  restart_script TEXT (重启脚本，可选)
  jvm_opts TEXT (JVM启动参数)
  env_vars TEXT (自定义环境变量，JSON格式)
  status TINYINT DEFAULT 1 (0=停用, 1=启用)
  create_time BIGINT
  update_time BIGINT
  索引: INDEX idx_status (status)

version_package 表 (版本包表)
  id BIGINT PRIMARY KEY AUTO_INCREMENT
  project_id BIGINT NOT NULL
  jar_name VARCHAR(200) NOT NULL
  file_path VARCHAR(500) NOT NULL (服务器上实际存储路径)
  file_size BIGINT (文件大小字节)
  version VARCHAR(50) NOT NULL (自动生成: 项目名-v版本号)
  sha256 VARCHAR(64) (文件SHA-256校验值)
  remark VARCHAR(500)
  create_time BIGINT
  索引: INDEX idx_project (project_id), INDEX idx_version (project_id, version)

deploy_record 表 (部署记录表)
  id BIGINT PRIMARY KEY AUTO_INCREMENT
  project_id BIGINT NOT NULL
  version_id BIGINT NOT NULL
  node_id BIGINT NOT NULL
  status TINYINT DEFAULT 0 (0=进行中, 1=成功, 2=失败, 3=回滚)
  jar_name VARCHAR(200)
  log TEXT (部署日志)
  start_time BIGINT
  end_time BIGINT
  create_time BIGINT
  索引: INDEX idx_project (project_id), INDEX idx_status (status)

alarm_record 表 (告警记录表)
  id BIGINT PRIMARY KEY AUTO_INCREMENT
  project_id BIGINT
  node_id BIGINT
  type VARCHAR(50) (节点离线/进程宕机/部署失败)
  content TEXT
  send_result TINYINT (0=失败, 1=成功)
  send_time BIGINT
  create_time BIGINT
  索引: INDEX idx_type (type), INDEX idx_create (create_time)

sys_user 表 (系统用户表)
  id BIGINT PRIMARY KEY AUTO_INCREMENT
  username VARCHAR(50) UNIQUE NOT NULL
  password VARCHAR(128) NOT NULL (BCrypt加密)
  role VARCHAR(20) DEFAULT 'operator' (admin=管理员, operator=操作员)
  status TINYINT DEFAULT 1 (0=禁用, 1=启用)
  create_time BIGINT
  update_time BIGINT
  索引: UNIQUE KEY uk_username (username)

operation_log 表 (操作审计表)
  id BIGINT PRIMARY KEY AUTO_INCREMENT
  user_id BIGINT NOT NULL
  module VARCHAR(50) (模块名)
  action VARCHAR(100) (操作动作)
  content TEXT (操作详情)
  ip VARCHAR(50) (操作来源IP)
  create_time BIGINT
  索引: INDEX idx_user (user_id), INDEX idx_module (module), INDEX idx_create (create_time)

file_access_log 表 (文件访问审计表，新增)
  id BIGINT PRIMARY KEY AUTO_INCREMENT
  user_id BIGINT NOT NULL
  node_id BIGINT NOT NULL
  file_type VARCHAR(20) (yml/log/jar)
  file_path VARCHAR(500) (服务器上的文件路径)
  action VARCHAR(20) (view/edit/download)
  content_summary VARCHAR(500) (摘要，编辑时记录变更)
  ip VARCHAR(50)
  create_time BIGINT
  索引: INDEX idx_node (node_id), INDEX idx_type (file_type), INDEX idx_create (create_time)

## 三、API 接口详细设计

Server 端 REST 接口：

用户登录
  POST /api/auth/login
  请求体: { username: string, password: string }
  响应: { code, message, data: { token, username, role } }
  说明: 验证用户凭据，返回 JWT Token，Token 存入 Redis 或内存，有效期24小时

节点管理接口：
  GET /api/nodes (节点列表，支持分页和状态筛选)
  GET /api/nodes/{id} (节点详情)
  POST /api/nodes (新增节点，填写name/ip/token/端口)
  PUT /api/nodes/{id} (修改节点信息)
  DELETE /api/nodes/{id} (删除节点，需确保无项目绑定)

版本包管理接口：
  POST /api/versions/upload (上传Jar包，multipart/form-data)
    参数: file (Jar文件), projectId (项目ID), remark (备注)
    响应: { version: string, filePath: string }
  GET /api/versions?projectId=xxx (版本列表)
  GET /api/versions/{id} (版本详情)
  DELETE /api/versions/{id} (删除版本)

项目管理接口：
  GET /api/projects (项目列表)
  GET /api/projects/{id} (项目详情)
  POST /api/projects (创建项目，绑定节点，配置脚本)
  PUT /api/projects/{id} (修改项目配置)
  DELETE /api/projects/{id} (删除项目)

部署发布接口：
  POST /api/deploy (选择版本发布)
    请求体: { projectId: long, versionId: long }
    说明: 将Jar包下发到所有绑定节点，自动备份旧版，停止进程，启动新版
  GET /api/deploy?projectId=xxx (部署记录列表)
  POST /api/deploy/{id}/rollback (指定部署回滚)

进程运维接口：
  POST /api/process/{projectId}/{nodeId}/start (启动)
  POST /api/process/{projectId}/{nodeId}/stop (停止)
  POST /api/process/{projectId}/{nodeId}/restart (重启)

日志查看接口：
  GET /api/logs/console?projectId=xxx&nodeId=xxx (实时控制台，WebSocket)
  GET /api/logs/file?projectId=xxx&nodeId=xxx&path=xxx&page=1&pageSize=100 (分页文件日志)

文件管理接口 (M10 新增)：
  GET /api/files/log?nodeId=xxx&path=xxx&page=1&pageSize=100
    说明: 读取指定节点指定路径下的日志文件内容，分页返回
  GET /api/files/config?nodeId=xxx&path=xxx
    说明: 读取指定节点指定路径下的YML配置文件内容
  POST /api/files/config?nodeId=xxx&path=xxx
    说明: 保存YML配置文件内容到指定节点指定路径 (先读取当前文件，比较差异后写入)
  POST /api/files/batch-download
    请求体: { items: [{ nodeId, path, type: yml|log|jar }] }
    说明: 将指定节点上的指定文件打包为Zip返回下载

监控接口：
  GET /api/monitor/process?projectId=xxx&nodeId=xxx
    响应: { status, pid, port, cpu, memory }

告警接口：
  GET /api/alarms?projectId=xxx (告警列表，支持分页筛选)
  GET /api/alarms/config (告警配置)
  PUT /api/alarms/config (配置SMTP服务器信息)

系统管理接口：
  GET /api/users (用户列表)
  POST /api/users (新增用户)
  PUT /api/users/{id} (修改用户)
  DELETE /api/users/{id} (删除用户)
  GET /api/users/me (当前用户信息)
  GET /api/operations?module=xxx&page=1 (操作审计日志)

## 四、Agent 端接口详细设计

Agent 对外暴露以下接口 (仅 Server 可访问)：

心跳接口：
  GET /api/node/heartbeat
  请求头: X-Token: <token>
  响应: { code, data: { nodeId, projects: [...] } }
  说明: 每30秒调用一次，上报节点基础信息 (OS版本/JDK版本/CPU/内存)

进程状态接口：
  GET /api/process/{projectId}/status
  响应: { status, pid, port, cpu, memory }
  说明: 查询指定项目的进程状态

文件读取接口 (M10 新增)：
  GET /api/files/read?path=xxx
    请求头: X-Token: <token>
    响应: { code, data: { content: string, fileName: string } }
    说明: 读取服务器上指定路径的文件内容，返回文本 (用于日志查看和YML读取)
  POST /api/files/write?path=xxx
    请求头: X-Token: <token>
    请求体: { content: string }
    说明: 将内容写入指定路径文件 (用于YML配置保存)
  GET /api/files/download?path=xxx
    请求头: X-Token: <token>
    响应: 文件流
    说明: 返回服务器上指定路径的文件，用于手动下载

日志接口：
  GET /api/log/tail?projectId=xxx&page=1&pageSize=200
    响应: { lines: string[] }
    说明: 分页读取项目运行日志文件
  GET /api/log/console?projectId=xxx&follow=true
    说明: WebSocket 连接，实时推送控制台输出

文件接收接口 (Server → Agent)：
  POST /api/files/receive
    请求头: X-Token: <token>
    请求体: multipart/form-data (Jar文件)
    响应: { code, message, filePath }
    说明: 接收Server下发的Jar包，保存到项目目录

## 五、核心业务流程设计

### 5.1 版本发布流程

1. 用户在前端选择项目和目标版本；
2. Server 校验项目状态和版本存在性；
3. Server 从版本存储区找到Jar文件；
4. Server 遍历项目绑定的所有节点；
5. 对每个在线节点：
   - 调用 Agent 的 /api/files/receive 接口上传Jar；
   - 调用 Agent 的停止脚本接口；
   - 等待进程完全退出；
   - 调用 Agent 的启动脚本接口启动新版；
   - 检测启动结果，记录部署日志；
6. 全部节点完成后更新项目状态；
7. 任一节点失败时记录失败原因，支持重试单个节点。

### 5.2 文件管理流程 (M10)

日志查看：
1. 用户选择节点，输入文件路径 (如 /data/logs/app.log)；
2. Server 通过 Agent 的 /api/files/read 接口读取文件内容；
3. 按分页参数截取对应行，返回前端；
4. 前端使用行号组件逐行显示，支持自动刷新。

YML配置编辑：
1. 用户选择节点，输入YML文件路径 (如 /data/app/application.yml)；
2. Server 调用 Agent 的 /api/files/read 接口读取当前内容；
3. 前端使用代码编辑器 (CodeMirror) 展示内容；
4. 用户修改后提交，Server 通过 /api/files/write 接口写入新内容；
5. 写入前做权限校验和路径安全检查 (禁止越权访问)；
6. 写入成功后记录操作审计。

批量下载：
1. 用户在前端勾选多个文件 (跨节点/跨类型)；
2. Server 逐个调用 Agent 的 /api/files/download 接口；
3. 将多个文件打包为 Zip 流返回前端；
4. 前端触发浏览器下载。

### 5.3 心跳保活流程

1. Agent 启动后每30秒向 Server 发送心跳请求；
2. Server 收到心跳后更新节点的 last_heartbeat 字段；
3. Server 定时任务 (每秒执行一次) 扫描所有节点；
4. 若节点超过 90 秒未收到心跳，判定为离线；
5. 节点离线时触发 M8 告警模块发送邮件通知。

  说明: Agent 通过此通道向Server推送实时控制台输出

  说明: 每10秒推送一次进程状态数据

## 七、安全设计

Token 认证：每个 Agent 有唯一 Token，每次 HTTP 请求携带 X-Token 头校验；
用户登录：BCrypt 密码加密，登录成功签发 JWT Token 存入内存；
文件安全：Server 侧校验 Agent Token，Agent 侧校验白名单路径 (禁止越权)；
前端安全：Token 存储于内存 (刷新失效)，所有请求通过 Axios 拦截器注入 Token；
XSS 防护：前端框架自动转义，后端参数化 SQL 查询；
路径校验：Agent 侧允许的路径前缀白名单配置，防止越权访问。

## 八、配置文件说明

Server application.yml 主要配置：
  server.port: 8081 (Web端口)
  server.path: ./data (数据目录)
  jpom.heart-second: 30 (心跳间隔秒数)

  server.url: http://server:8081 (Server地址)
  token: xxxxx (身份令牌)
  allow.path.prefix: /data,/app (允许访问的路径前缀)

## 九、定时调度设计

Server 侧调度任务 (使用 Quartz 或 Spring @Scheduled)：

1. 心跳检测任务：每秒执行，扫描 node_info 表，更新状态；
2. 定时重启任务：从 project_info 读取 Cron 表达式，调度对应项目重启；
3. 自动重启任务：检测进程离线状态，执行重启命令；
4. 日志清理任务 (可选)：清理过期日志文件。
