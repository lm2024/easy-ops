## 二、路由与菜单设计

前端菜单对应 8 个模块：

侧边菜单 (左侧导航栏，Ant Design Menu)：
  仪表盘 (Dashboard)
  节点管理 (node/list)
  版本管理 (version/list)
  项目管理 (project/list)
  部署管理 (deploy/list)
  进程运维 (process/console)
  文件管理 (file/log-view) (M10 新增)
  告警中心 (alarm/list)
  系统管理 (system/users)

路由表 (Vue Router 4 历史模式)：
  /login 登录页 (无需权限)
  /dashboard 仪表盘 (需登录)
  /nodes 节点列表 (需节点查看权限)
  /nodes/create 新增节点 (需节点编辑权限)
  /nodes/:id 节点详情 (需节点查看权限)
  /versions 版本列表 (需版本查看权限)
  /projects 项目列表 (需项目查看权限)
  /projects/create 创建项目 (需项目编辑权限)
  /projects/:id 项目详情 (需项目查看权限)
  /deploy 部署记录 (需部署查看权限)
  /process 进程运维 (需进程操作权限)
  /process/:id/console 控制台 (需进程操作权限)
  /process/:id/log 日志查看 (需日志查看权限)
  /process/:id/monitor 监控面板 (需监控查看权限)
  /files 文件管理 (需文件操作权限) (M10 新增)
  /files/log 日志文件查看 (M10 新增)
  /files/config 配置编辑 (M10 新增)
  /files/download 批量下载 (M10 新增)
  /alarms 告警中心 (需告警查看权限)
  /users 用户列表 (需系统管理权限)
  /operations 操作审计 (需系统管理权限)

路由守卫：
  未登录重定向到 /login；
  无权限路由跳转到 403 页面；
  所有路由加载时触发异步加载。

## 三、页面详细设计

### 3.1 布局框架 (MainLayout)

整体布局采用 Ant Design Layout 左右结构：
左侧：固定宽度 220px 的侧边菜单 (AppAside)，深色背景；
中间：白色背景内容区，可折叠；
顶部：固定高度 56px 的顶栏 (AppHeader)，显示用户名、退出按钮、系统信息。

### 3.2 登录页 (LoginPage)

居中卡片，包含用户名输入框、密码输入框、登录按钮；
登录成功后存储 Token 到内存，重定向到仪表盘。

### 3.3 节点列表页 (NodeList)

Ant Design Table 表格展示，列：节点名称、IP、端口、状态 (在线/离线)、JDK版本、最后心跳时间、操作 (编辑/删除/详情)；
支持按状态筛选，顶部搜索框按名称/IP搜索；
操作列：在线节点标记为绿色，离线节点红色高亮。

### 3.4 版本列表页 (VersionList)

表格展示项目名、Jar包名、版本号、文件大小、备注、上传时间；
支持按项目筛选，每个项目下有独立的版本列表；
操作列：查看详情、下载Jar包、删除。

### 3.5 项目列表页 (ProjectList)

卡片式展示，每个项目显示：项目名称、绑定节点数、运行状态、最后部署时间；
点击卡片进入项目详情页；
支持创建新项目、编辑项目配置、删除项目。

### 3.6 项目详情页 (ProjectDetail)

信息卡片：项目名称、JVM参数、启停脚本、环境变量；
部署记录列表：历史部署状态、时间、操作 (回滚)；
操作区：选择版本发布、节点启停按钮；
文件管理入口 (M10 新增)：跳转至该项目的文件管理页。

### 3.7 进程运维页面 (process)

控制台 (ProcessConsole)：
  顶部选择项目和节点；
  中间区域使用 xterm.js 终端组件展示实时控制台输出；
  支持发送命令 (可选)；
  实时连接通过 WebSocket 建立。

日志查看 (ProcessLog)：
  选择项目、节点后，分页读取服务器日志文件；
  使用 LogViewer 组件，带行号显示；
  支持日志关键字搜索和高亮。

监控面板 (ProcessMonitor)：
  使用 ECharts 展示 CPU、内存使用趋势图；
  实时通过 WebSocket 获取数据，自动刷新。

### 3.8 告警中心 (alarm)

告警历史列表：时间、项目/节点、告警类型、发送结果；
告警配置页：SMTP服务器、端口、发件邮箱、密码、告警接收人列表。

### 3.9 系统管理 (system)

用户列表：用户名、角色、状态、操作；
用户创建：用户名、密码、角色选择；
操作审计：操作时间、模块、操作人、操作内容。

### 3.10 文件管理页面 (M10 新增)

文件日志查看 (FileLogView)：
  选择节点后，输入服务器文件路径 (如 /data/logs/app.log)；
  使用 LogViewer 组件分页展示日志文件内容；
  支持关键字搜索、自动刷新 (定时轮询)；
  路径输入支持通配符 (如 /data/logs/*.log)。

YML 配置编辑 (FileConfigEdit)：
  选择节点后，输入YML文件路径 (如 /data/app/application.yml)；
  使用 CodeEditor 组件加载并展示YML内容；
  支持语法高亮 (YAML 语法)；
  编辑后点击保存，通过后端接口写入服务器；
  保存前显示变更预览 (Diff 视图)；
  保存失败时提示错误信息。

批量下载 (FileDownload)：
  选择节点，勾选需要下载的文件；
  文件类型：日志、YML配置、Jar包；
  支持跨节点批量选择；
  点击"批量下载"按钮，后端打包为 Zip 流返回；
  前端触发浏览器下载，显示下载进度。

## 四、组件设计

公共组件：

AppTable (封装表格)：
  属性：columns, dataSource, loading, pagination, scroll, rowKey；
  插槽：custom 操作列；
  默认支持分页、排序、筛选。

AppModal (封装弹窗)：
  属性：title, visible, width, confirmLoading, okText, cancelText；
  插槽：default 内容区。

CodeEditor (封装 CodeMirror)：
  属性：value, readOnly, mode (yaml/java/log)；
  支持语法高亮、行号显示、代码折叠。

LogViewer (日志查看器)：
  属性：lines, loading, autoScroll, keyword；
  带行号的文本展示，支持关键字高亮，支持自动滚动到底部。

Terminal (终端组件)：
  属性：connected, command；
  封装 xterm.js，支持 WebSocket 实时输出。

StatusBadge (状态标签)：
  属性：status (online/offline/stopping/starting/running)；
  显示带颜色的状态标签。

## 五、接口封装 (Axios 拦截器)

request.ts 封装 Axios 实例：
  基础URL：/api (Nginx 反向代理)；
  请求拦截器：注入 Authorization: Bearer <token>；
  响应拦截器：统一处理错误码，401 跳转登录；
  超时时间：30 秒 (文件传输接口 5 分钟)。

## 七、前端安全设计

Token 存储于内存 (不持久化到 localStorage，刷新失效)；
登录页密码使用 SHA-256 传输加密；
文件路径校验 (前端限制路径字符集，防止注入)；
XSS 防护 (Vue 自动转义插值表达式)；
CSRF 防护 (Token 存储在内存中，Cookie 标记 HttpOnly)。
