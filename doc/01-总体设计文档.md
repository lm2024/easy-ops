# 一体云运维平台 - 总体设计文档 (v1.0)

## 一、项目概述

面向内网离线环境，构建轻量级 Server-Agent 分布式运维平台，实现 SpringBoot 微服务 Jar 包统一上传、多服务器集群批量部署、进程托管与日常运维。

**环境约束**：JDK 8、纯内网运行、无 Docker、禁止标准 CI/CD 工具。

**设计思想**：参考分布式进程管理框架的设计模式——服务端集中管理，Agent 端轻量执行；WebSocket 实时推送，HTTP 双向通信；Plugin 化插件扩展；Storage 抽象存储层。在此基础上按互联网最佳实践进行重构，代码结构、命名规范、架构分层全部自主设计，不复用任何第三方方案的既有实现。

**发布模式**：人工本地上传 Jar 包 → 自动版本编号 → 集群批量分发 → 自动备份 → 停旧启新 → 一键回滚。

## 二、架构设计

### 2.1 整体架构

```
┌─────────────────────────────────────────────────┐
│                    运维人员                       │
└────────────────────┬────────────────────────────┘
                     │ HTTPS / WSS
                     ▼
┌─────────────────────────────────────────────────┐
│                   Server 端                      │
│  ┌──────────┐ ┌──────────┐ ┌──────────────────┐ │
│  │ Web 管理  │ │ RESTful  │ │ WebSocket 服务   │ │
│  │ 前端页面  │ │ API 服务 │ │ (实时推送/日志)  │ │
│  └──────────┘ └──────────┘ └──────────────────┘ │
│  ┌─────────────────────────────────────────────┐ │
│  │  业务层：节点管理 / 项目管理 / 版本管理       │ │
│  │  告警服务 / 定时调度 / 自动重启               │ │
│  └─────────────────────────────────────────────┘ │
│  ┌─────────────────────────────────────────────┐ │
│  │  存储层：H2 嵌入式数据库 (单文件)             │ │
│  └─────────────────────────────────────────────┘ │
└──────┬────────────┬────────────┬────────────────┘
       │ HTTP+Token │ HTTP+Token │ HTTP+Token
       ▼            ▼            ▼
┌──────────┐ ┌──────────┐ ┌──────────┐
│  Agent 1 │ │  Agent 2 │ │  Agent N │
│ 服务端管  │ │ 服务端管  │ │ 服务端管  │
│ 理端程序  │ │ 理端程序  │ │ 理端程序  │
└──────────┘ └──────────┘ └──────────┘
```

### 2.2 分层架构

**Server 端** 采用经典三层架构：
- **表现层 (Web)**：Vue 3 SPA 前端 + Spring Boot REST API
- **业务层 (Service)**：实体业务逻辑、流程编排
- **数据层 (Storage)**：H2 嵌入式数据库，单文件持久化

**Agent 端** 轻量后台程序：
- **命令层 (Commander)**：进程启停、日志读取、状态采集
- **通信层 (Transport)**：HTTP 上传/下载、WebSocket 实时通信
- **守护层 (Daemon)**：心跳保活、定时任务、自动重启

### 2.3 通信架构

| 通信方式 | 方向 | 用途 | 协议 |
|---------|------|------|------|
| HTTP POST | Server → Agent | 文件下发、命令执行 | HTTP 1.1 + Token |
| WebSocket | Server ↔ Agent | 实时日志/控制台/状态推送 | WSS (内网可 WS) |
| 心跳请求 | Agent → Server | 节点在线状态检测 | HTTP GET |

## 五、通信协议设计

### 5.1 认证机制

- 每个 Agent 配置 Token（32 位随机字符串），注册时写入数据库；
- 每次 HTTP/WebSocket 请求 Header 携带 `X-Token: <token>` 进行身份校验；
- Server 侧对 Token 进行一致性校验，拒绝非法请求。

### 5.2 API 规范

- RESTful 风格 URI，动词为名词 + HTTP 方法；
- 统一响应格式：`{ code: number, message: string, data: any }`；
- 文件上传使用 multipart/form-data，下载使用流式响应；
- WebSocket 消息格式：`{ type: string, seq: number, data: any }`。

### 5.3 文件传输协议

- 大包分片传输（单片 1MB），Server 端接收完成后合并校验；
- 断点续传：记录已传输分片，支持中断后续传；
- 传输完成后 SHA-256 校验确保文件完整性。

## 六、数据库设计

### 6.1 核心表结构

| 表名 | 说明 | 核心字段 |
|------|------|---------|
| node_info | 节点信息 | id, name, ip, port, token, status, jdk_version |
| project_info | 项目管理 | id, name, node_ids, start_script, stop_script, jvm_opts |
| version_package | 版本包 | id, project_id, jar_name, file_path, version, remark |
| deploy_record | 部署记录 | id, project_id, version_id, node_id, status, log |
| alarm_record | 告警记录 | id, project_id, node_id, type, content, send_result |
| sys_user | 系统用户 | id, username, password, role, status |
| operation_log | 操作审计 | id, user_id, module, action, content, create_time |

### 6.2 数据库选型

采用嵌入式 H2 数据库，单文件持久化（默认路径 `{app.data}/data.mv.db`），无需安装任何外部数据库服务。H2 支持 SQL 标准，可通过 Web 控制台在线查看。备份功能支持一键导出完整数据库文件。

## 七、前端架构设计

### 7.1 技术选型

基于 Vue 3 Composition API + TypeScript 构建单页应用，使用 Ant Design Vue 4 作为企业级 UI 框架。路由采用 Vue Router 4 历史模式，状态管理使用 Pinia。

### 7.2 目录结构

```
web/
├── src/
│   ├── api/          # 接口层 (按模块分文件)
│   ├── components/   # 公共组件 (表格/弹窗/表单等)
│   ├── views/        # 页面 (按模块分目录)
│   ├── stores/       # Pinia 状态管理
│   ├── router/       # 路由配置
│   ├── types/        # TypeScript 类型定义
│   ├── utils/        # 工具函数
│   └── assets/       # 静态资源
├── public/           # 静态资源
├── index.html        # 入口 HTML
├── vite.config.ts    # Vite 配置
└── package.json
```

### 7.3 安全架构

- 前端：HTTPS 传输，Token 存储于内存（刷新失效），CSS 变量主题系统；
- 后端：Token 认证、SQL 注入防护（MyBatis 参数化）、XSS 防护、文件类型白名单校验；
- Agent：Token 固定不可变，通信白名单。

## 八、部署架构

### 8.1 Server 部署

```
启动：java -jar server.jar --server.port=8081 --server.path=./data
配置文件：application.yml (端口/JVM/邮件 SMTP 配置)
数据库：./data/data.mv.db (自动创建维护)
日志：./logs/
```

### 8.2 Agent 部署

```
启动：java -jar agent.jar --server.url=http://server:8081 --token=xxx
配置文件：application.yml (Server 地址/Timing 参数)
```

### 8.3 系统架构特征

- Server 和 Agent 均为独立 JAR 包，零外部依赖；
- 无需安装任何数据库、消息队列、缓存中间件；
- 一台服务器部署一个 Server 节点，Agent 部署在各业务服务器上；
- 全部端口可配置，支持防火墙穿透。
